1. ARCHITECTURE (High-Level System Design)
This is the structure Claude will actually code against.
A. Conversation Stages (Invisible to the User)
The engine moves fluidly through these stages based on user responses:
1. Warm Start

2. Recruiter Screen

3. Hiring Manager Deep Dive

4. Behavioral & Competency

5. Strategy / Critical Thinking

6. Leadership & Executive Presence (only if applicable)

7. Wrap-Up

8. Automated Debrief (post-interview)

The stages are not announced. Flow is natural, based on detection of gaps.
________________


B. Signal Tracking System
Every answer the user gives is tagged with signals:
   * Background Context

   * Functional Competency

   * Leadership

   * Collaboration

   * Ownership

   * Strategic Thinking

   * Problem Solving

   * Communication Clarity

   * Metrics Orientation

   * Stakeholder Management

   * Executive Presence (if applicable)

The simulator “collects” these like badges.
When enough signals are met, it transitions.
________________


C. Core Components
1. Intent Classifier
Determines what type of answer the user gave.
If lacking depth → follow-up.
If complete → move forward.
2. Question Generator
Selects next question based on:
      * Which signals are missing

      * Role seniority

      * Story quality

      * Time elapsed

      * Whether the candidate sounds stuck

3. Follow-Up Engine
Triggers automatically when the candidate gives:
         * Vague stories

         * Missing metrics

         * No ownership

         * No “I” statements

         * Ambiguous outcomes

         * Overly polished corporate answers

4. Wrap Trigger
When all required signal categories hit a threshold → shift to wrap-up.
________________


2. DECISION LOGIC (How It Flows Like a Human)
This is the logic Claude will implement in the backend prompt.
________________


A. Warm Start → Recruiter Questions
Trigger: always first.
User answers → classifier tags for:
            * Role fit

            * Motivation

            * Compensation

            * Experience alignment

If motivation unclear → ask follow-up.
If fine → transition.
________________


B. Recruiter → Hiring Manager
Trigger conditions:
               * User has provided clear background

               * Role interest captured

               * High-level experience acknowledged

Then the engine says something like:
“Great, that gives me helpful context. Let’s go deeper into your experience.”
Now it switches to functional depth.
________________


C. Hiring Manager → Behavioral
Trigger:
                  * Functional basics established

                  * One or two strong examples provided

                  * Gaps in leadership, conflict, influence, or delivery

Behavioral questions arise naturally from what the user already said:
“You mentioned leading a complex project. Tell me about a time things went off-track and how you handled it.”
No hard module switch.
________________


D. Behavioral → Strategy / Problem Solving
Trigger:
                     * Candidate has demonstrated some leadership and delivery

                     * No evidence yet of strategic thinking

                     * Role seniority signals strategy is required

Engine transitions like this:
“Thanks for sharing that. I’d like to understand how you think about bigger problems.”
Then moves into scenario questions.
________________


E. Strategy → Executive Presence (Optional)
Trigger if:
                        * Candidate is Director+

                        * Answers show leadership

                        * They mention presenting to execs

OR

                        * It’s a senior role by job family

Engine prompts:
“Imagine you’re presenting this plan to a skeptical COO. How do you communicate it?”
________________


F. Wrap-Up
Trigger:
All required signals collected OR time threshold met.
Engine asks:
“What questions do you have for me?”
“Anything you’d like to clarify?”
Smooth ending.
________________


G. Post-Interview Debrief
After wrap-up, the engine generates:
                           * Strengths

                           * Weak signals

                           * Opportunities

                           * Leveling estimate

                           * Competency-specific coaching

                           * A recommended practice plan

This happens after the interview to preserve realism.
________________

ADDITION TO THE EXISTING INTERVIEW SYSTEM FOR CLAUDE
(Role-Specific Question Integration)
________________


NEW REQUIREMENT: ROLE-SPECIFIC INTERVIEW QUESTIONS
Purpose:
Enhance the conversational interview simulator so that questions automatically adapt to the user’s target role (Product Manager, UX Designer, UI Designer, Software Engineer) without creating separate modules.
How It Fits Into the Current System:
Role-specific questions should be woven into the existing conversational flow. The module structure stays exactly the same. Only the question source changes.
________________


IMPLEMENTATION RULES
1. Use Role-Specific Questions Starting in the Hiring Manager Deep Dive
When the conversation reaches the Hiring Manager phase, the system should pull questions from the appropriate role-specific question bank.
Example:
If target_role = Product Manager → use PM deep-dive questions.
If target_role = SWE → use SWE deep-dive questions.
If no role is specified → default to general leadership/functional questions.
________________


2. Behavioral Questions Become Role-Aware
Use the same behavioral themes (conflict, leadership, influence, ownership),
but substitute role-aligned examples.
Examples:
                                                                                                   * PM: “Tell me about a time engineering pushed back on your roadmap.”

                                                                                                   * UX: “Tell me about a time research changed your design direction.”

                                                                                                   * SWE: “Tell me about a technical disagreement you had with a teammate.”

If unclear → fallback to general behavioral questions.
________________


3. Strategy / Critical Thinking Becomes Role-Specific
Insert strategy questions that match the role.
Examples:
                                                                                                      * PM: prioritization, roadmap cuts, product metrics

                                                                                                      * UX: research constraints, usability problems, persona conflicts

                                                                                                      * UI: accessibility, visual hierarchy, systems consistency

                                                                                                      * SWE: scaling systems, debugging, architecture tradeoffs

This replaces the generic strategy question for that stage.
________________


4. Leadership / Executive Presence Becomes Role-Specific (Senior Roles Only)
When the simulator detects seniority (director+ signals or user profile),
use leadership questions tailored to that discipline.
Examples:
                                                                                                         * PM: “Explain your product strategy to a skeptical VP.”

                                                                                                         * SWE: “Communicate a system migration plan to non-technical leaders.”

                                                                                                         * UX: “Justify a design direction to executives who want data.”

If user is mid-level → skip executive presence questions.
________________


5. All Other Stages Remain the Same
Warm start, recruiter screen, wrap-up, and debrief do not change.
Only the content of mid-stage questions adapts.
