CSS Animations

.chat-widget {
  position: fixed;
  bottom: 24px;
  right: 24px;
  cursor: pointer;
  z-index: 1000;
  transition: all 0.3s ease;
}

/* Noticeable breathing animation (15% scale) */
@keyframes breathe {
  0%, 100% {
    transform: scale(1);
  }
  50% {
    transform: scale(1.15);
  }
}

.breathe {
  animation: breathe 2.5s ease-in-out infinite;
}

/* Add glow on hover */
.chat-widget:hover svg {
  filter: drop-shadow(0 0 12px rgba(102, 126, 234, 0.6));
}

/* Active state when chat is open */
@keyframes pulse-active {
  0%, 100% {
    transform: scale(1);

  }

.chat-widget.active svg {

JavaScript Toggle

function openChat() {
  const widget = document.querySelector('.chat-widget');
  const svg = widget.querySelector('svg');
  
  // Toggle active state
  widget.classList.toggle('active');
  
  // Switch to active animation when chat opens
  if (widget.classList.contains('active')) {
    svg.classList.remove('breathe');
    svg.classList.add('active-state');
  } else {
    svg.classList.remove('active-state');
    svg.classList.add('breathe');
  }
  
  // Your chat open/close logic here
  toggleChatWindow();
}

Design Rationale

Why Breathing Works

  • → Organic, not mechanical
  • → Signals availability without being pushy
  • → Catches attention in peripheral vision
  • → Reinforces "human + AI" positioning

Why Not Generic Bubble

  • → Looks like every other support widget
  • → Doesn't reinforce your brand
  • → Signals "generic chatbot," not "Henry"
  • → Missed opportunity to differentiate
Pro tip: When the chat is open, switch to the "active state" animation (faster pulse with glow). This gives visual feedback that the system is listening and ready to respond.
